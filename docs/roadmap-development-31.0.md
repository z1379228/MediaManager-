# Development 31.0 版本計畫

> 收斂狀態（2026-07-18）：`DONE / SUPERSEDED`。本文件以下內容為歷史規劃；唯一 canonical 計畫為 `roadmap-development-34.0-40.0.md`。

歷史狀態（封存時）：規劃中。31.0 延續 30.0 的驗證基線，專注於「參考軟體差異整理、可診斷的更新計畫與跨 MOD 一致性」，不新增繞過登入、DRM、Cookie、Cloudflare、廣告或付費限制的能力。

## P0：規劃與安全基線

- **參考軟體差異表**：將下載器、媒體庫、轉換器與 P2P 參考功能映射到現有 MOD／核心邊界，標示已實作、候選與明確排除項。
- **版本計畫可追溯性**：每個 P0/P1/P2 項目具備來源、驗收條件、阻擋原因與目前狀態，避免排程只重複執行稽核。
- **MOD 父子一致性**：父 MOD、子 MOD、四語言顯示與不可用／製作中狀態維持相同安全判定。
- **證據完整性**：維持 Ruff、回歸測試、網站／MOD 稽核、文字污染掃描與日常版本雜湊稽核；完整歷史稽核仍不得納入使用者資料。

## P1：功能與診斷優化

- YouTube／Music、Bilibili、動畫瘋、MEGA 與社群官方頁的功能差異與錯誤分類文件化，讓下一版能按網域分流驗證。
- 下載佇列、預覽／縮圖、取消／重試與重啟狀態的診斷訊息統一，維持增量更新與原子輸出。
- 依賴診斷表補充 yt-dlp EJS、MEGAcmd、whisper-cli 與語音模型的可選安裝狀態；不自動下載或執行未知安裝程式。
- 失效影片替代搜尋、歷史紀錄與匯出報告維持低資源、使用者觸發與可回溯設計。

## P2：條件式候選

- 只有在官方公開、非登入、非 DRM 的端點可驗證時，才評估新的網站下載子 MOD。
- Gopeed、P2P／Torrent 與通用解析器維持獨立安全邊界，不進入核心或預設啟用。
- 參考軟體新增功能僅先列入候選，不因比較結果自動改變安全策略或發布通道。

## 每輪純規劃驗收

1. 唯讀掃描工作樹、文件、參考清單、版本與 MOD 狀態。
2. 建立或補充下一個未完成版本計畫；若無新計畫差異，記錄原因並排入下一輪更新項目。
3. 可執行唯讀的 Ruff、相關測試、網站品質、MOD 群組、版本與文字污染稽核，只記錄結果。
4. 不修改程式、不刪除使用者資料、不安裝軟體、不打包、不提交、不推送；外部限制保留為阻擋項。

31.0 不建立 Stable 版本；是否建立 Testing 候選須另行取得使用者明確決定。
