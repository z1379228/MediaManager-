# Development 38.0 版本計畫

> 重新設計狀態（2026-07-18）：`SUPERSEDED`。本文件以下內容只保留歷史草案；38.0 的唯一執行定義與優先序改由 [34.0～40.0 主計畫](roadmap-development-34.0-40.0.md) 管理。

歷史草案主題（`DO NOT EXECUTE`）：38.0 原聚焦「參考軟體差異驗證、資源上限與可回溯效能資料」；內容未依新優先序重寫，不得作為目前規格。

## P0：資源與安全

- **資源上限**：縮圖、預覽、並行工作、暫存空間與歷史紀錄維持明確上限，超限時顯示可操作原因。
- **取消與清理**：背景工作可取消、停止後釋放執行緒與檔案控制代碼，異常時保留可恢復診斷。
- **參考功能對照**：將同類軟體的批量、佇列、格式與通知功能逐項標記已實作、候選或安全排除。
- **驗證證據**：Ruff、回歸測試、網站／MOD／版本／文字污染稽核保持可重現；UserData 不進版本雜湊。

## P1：效能與可觀測性

- YouTube／Music、Bilibili、動畫瘋與 MEGA 工作區採增量刷新，避免大清單重複建立 UI 元件。
- 下載、搜尋與預覽記錄耗時、佇列長度與失敗類別的脫敏摘要，供使用者匯出診斷。
- UI 在 940×620 最小尺寸、深色主題與鍵盤操作下維持可用，長標題與縮圖載入不破圖。
- 依賴檢查顯示版本、路徑與可用性，不在背景下載或執行未信任工具。

## P2：條件式候選

- 快取、並行與預取僅在使用者明確啟用且有上限時評估。
- 新網站、Gopeed、P2P／Torrent 與通用解析器維持候選，不進入核心。
- 雲端同步、帳號與廣告移除功能不列入本機免費核心。

## 每輪純規劃驗收

1. 唯讀掃描工作樹、文件、參考清單、版本與 MOD 狀態。
2. 建立或補充下一個未完成版本計畫；若沒有新差異，記錄原因與下一輪候選。
3. 只記錄唯讀稽核結果，不修改程式碼、使用者資料或版本產物。
4. 外部網站限制、正式簽章缺失與歷史 UserData 項目維持為阻擋項。

38.0 不建立 Stable 或 Testing 產物；發布需另行取得使用者明確決定。
