# 開發版 13.0～14.0 實際更新紀錄

本文件集中記錄 13.x 到 14.0 的實際完成內容。未完成項目不會寫成已支援；正式版仍須另行評估並取得使用者確認。

## 13.0

- 新增動畫瘋離線保存子 MOD：只保存使用者明確選取集數的公開中繼資料、封面與使用者自行提供的本機媒體，不解析串流、不繞過廣告、DRM、登入或地區限制。
- 動畫瘋父／子 MOD、四語言資源、可信 UI 與自檢接線完成。
- 建立全新的 `Version/Development/13.0`，未覆寫既有版本；813 項測試通過，2 項 Windows symlink 測試略過。
- 開發版維持 `SAFE_MODE`；production Ed25519 身分與 Authenticode 簽署仍是正式發行阻擋。

## 13.1

- 公開 provider smoke 報告升級至 schema v2，逐案記錄嘗試次數、耗時、錯誤與失敗類別。
- 只對 HTTP 408／425／429／5xx、連線重設或逾時等 `temporary-upstream` 狀態進行最多三次的有界重試；登入、地區、移除內容、本機依賴或契約錯誤不重試。
- 暫時上游錯誤即使完成重試仍保持失敗，不會把 Dailymotion HTTP 502 或其他外部服務異常偽裝成全綠。
- 報告新增通過、失敗與暫時上游失敗摘要，方便分辨網站波動和本機／程式回歸。

## 13.2

- YouTube 單片下載工作區會顯示受限 HTTPS CDN 縮圖；從搜尋結果帶入和重新讀取網址資訊都使用同一條可信縮圖載入流程。
- 單片網址只啟用讀取資訊、30 秒試聽與選用的 60 秒影片預覽；純播放清單才啟用展開按鈕，帶有播放清單內容的單片網址同時保留兩種明確操作。
- 搜尋結果與播放清單仍提供縮圖、試聽、停止與選用影片預覽；切換選取、停止、關閉或失敗都會清除預覽暫存。

## 13.3

- 重新核對播放清單選取、批量佇列、暫停／恢復／取消與重啟狀態：展開後預設不選取，必須由使用者明確勾選才加入佇列。
- 等待中或執行中的工作在重新開啟程式後保持暫停，不會因乾淨啟動自動開始網路下載；只有使用者按下恢復才重新排入佇列。
- 長音訊切割維持先產生章節／靜音候選、預覽及人工確認，再建立有界音訊批次；未確認草案不能送入下載佇列。

## 13.4

- Bilibili 搜尋維持獨立來源、官方縮圖、UP 主篩選／鎖定與多選帶入，不會回退到 YouTube。
- 單片下載工作區新增 Bilibili 官方 CDN 縮圖；多分段優先讀取官方 view API 的 `part`、長度與首幀，缺少欄位時才使用父標題加 P 編號，不再只顯示未命名分段。
- 彈幕維持獨立子 MOD：可選 XML、ASS 與 MKV 封裝；轉換或封裝失敗時保留既有影片與可用 sidecar，不把局部失敗誤報成整體成功。
- 佇列、切割、Bilibili provider／工作區與 UI 目標回歸共 97 項通過。

## 13.5

- 動畫瘋番劇儲存新增按需完整性驗證；只在使用者按下按鈕後核對離線索引、檔案大小與 SHA-256，不在啟動時掃描大型媒體。
- 本機媒體匯入改為明確的「匯入／重新連結」流程；缺檔、大小不符或雜湊不符會分開回報，可重新選取使用者已有的本機媒體修復索引。
- 驗證可取消、拒絕 symlink／越界路徑／不合法中繼資料，檔案在核對期間發生變更也會停止，不會把不穩定結果寫成通過。
- 繁中、簡中、英文、日文 UI 與內建 MOD LF 完整性雜湊同步；37 項相關測試通過，1 項 Windows symlink 測試略過。

## 13.6

- Facebook 收斂為公開影片網址＋完整影片下載：UI 不再顯示音訊轉出、字幕、分段或播放清單選項，provider 也會拒絕繞過可信 UI 傳入的非影片／部分下載參數。
- Facebook 仍只接受有界的官方 watch、reel、videos 與 `fb.watch` 公開網址，顯示經允許 CDN 驗證的縮圖；無法解析時明確說明公開頁與 `curl-cffi` 條件，不讀取瀏覽器 Cookie。
- MEGA 保持獨立檔案工作區，辨識公開檔案／資料夾與影片、音訊、壓縮檔、文件等輸出類型；不混入影音格式、字幕、試聽或播放清單控制。
- MEGA 自訂連線數／速率只在偵測到官方 `mega-get`／`mega-speedlimit` 後啟用，資料夾完成後核對本機樹狀輸出；66 項相關測試通過，1 項 Windows symlink 測試略過。

## 13.7

- 網站搜尋維持一次選擇一個明確來源：YouTube／YouTube Music、Bilibili 與動畫瘋各走自己的搜尋子 MOD，指定來源失敗時不改查其他網站。
- 主 MOD 未啟用時不列出其子搜尋，來源摘要會說明啟用條件；啟用後即時加入來源清單，切換來源會使舊分頁游標失效。
- 分頁游標綁定 provider、上限 2048 字元，跨頁結果依網站媒體身分去重並限制 200 筆；縮圖只接受可信 HTTPS CDN。
- 試聽與影片預覽只對宣告能力且可安全路由的 YouTube 結果啟用；Bilibili／動畫瘋不會誤用 YouTube 預覽 provider。46 項相關回歸通過。

## 13.8

- MOD SDK 不再把相容核心版本寫死為 3.0.0；新範本會依目前開發版填入 `minimum_core_version`／`maximum_core_version`。
- 一般 `create` 範本新增 schema v2 宣告式 `ui.json`，預先提供英文、日文、簡中與繁中；驗證時會檢查 UI 路徑、大小、結構與主動內容限制。
- 新增 `create-site`／`validate-site`：建立 data-only 網站父 MOD 與 `python-subprocess` 下載子 MOD，子層透過 dependency 綁定父層，離線檢查父子 ID、runtime protocol 1.0 與四語 UI。
- 範本採暫存目錄完成後再原子改名，不覆寫既有目錄，也不建立私鑰、`files.json`、簽章或假信任狀態；38 項 SDK／UI／安裝／受控執行回歸通過。

## 13.9

- 手動自檢新增動畫瘋離線封存驗證按鈕、匯入／驗證取消狀態與父子 MOD 實際顯示狀態檢查；自檢只讀取暖機後記憶體狀態，不刷新依賴、不啟動 provider。
- 冷啟動驗證確認可選下載、搜尋、轉換、轉錄及自動化 provider 都不建立子程序；下載佇列也不會因重新啟動自動繼續。
- 完整主視窗離屏組裝測試確認 940×620 最小尺寸、深色背景、YouTube／Bilibili 獨立工作區、四種核心語言與 MOD 管理入口能一起建立並安全關閉。
- 13.9 UI、自檢、冷啟動、入口與 MOD 管理針對性回歸共 53 項通過；封版仍以 14.0 全量 Ruff、Pytest、完整性與複製資料夾啟動結果為準。

## 14.0

- 核心與專案版本同步升為 14.0.0；全量 Ruff、825 項 Pytest 與 8 個網站父群組 × 4 語言稽核通過，另有 2 項 Windows symlink 權限測試明確略過。
- PowerShell 來源污染掃描通過 487 個 Git 追蹤／未追蹤文字檔；手動自檢、冷啟動、940×620 完整主視窗及所有內建 MOD 完整性測試通過。
- 建立全新的 `Version/Development/14.0`，未替換 13.0；共 109 個檔案、385,297,666 bytes，`SHA256SUMS.txt` 驗證 108 個成品檔案全部通過。
- copied-folder 的 `--version`、`--portable --verify-only`、`--portable --headless` 均為 exit 0；隱藏 GUI 啟動保持回應，測試副本與測試程序均已清除。
- 一次性 Ed25519 演練核對 106 個簽署檔、成功偵測竄改且未保留私鑰。正式 preflight 仍因缺 production Ed25519 key id／public key 與 Authenticode `NotSigned` 而阻擋，因此成品維持 Development／`SAFE_MODE`。
- 公開 provider smoke 為 6/7；YouTube、Vimeo、SoundCloud、TikTok、Twitch 與 Bilibili 通過，Dailymotion 固定樣本兩次皆為上游 HTTP 502，分類為 `temporary-upstream` 並維持 FAIL，不偽裝全綠。
- 本次沒有建立 Testing 或 Stable，也沒有上傳或覆寫 GitHub Release。
