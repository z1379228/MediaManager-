# MediaManager 開發版 5.1 至 5.9 更新簡介

依雙軌發布政策，5.1 至 5.9 不建立重複的大型封裝，只保留更新簡介；完整附件
集中在開發版 6.0。

| 開發版 | 完成內容 |
| --- | --- |
| 5.1 | 搜尋來源錯誤分成 timeout、invalid-response、unavailable 與一般錯誤；既有取消、預覽清理及 UI generation 防止過期回呼重新開啟畫面。 |
| 5.2 | YouTube 搜尋支援有界 offset 下一頁，每頁可選 12／20／30／50 筆；加入本機長度、內容類型、四語篩選與穩定去重排序。 |
| 5.3 | `youtube-history`、`youtube-recovery`、`youtube-similar` 維持獨立可停用 MOD；替代與相似候選上限提高到 50。 |
| 5.4 | 外部 MOD 宣告式 UI schema v2 支援 `en`、`ja`、`zh-CN`、`zh-TW`；缺少選定語言時回退到 MOD 預設語言，其他語言拒絕載入。 |
| 5.5 | 下載佇列沿用優先級、暫停、恢復、重啟復原及完成通知；預檢實際保留估計輸出容量與 256 MiB 安全空間。 |
| 5.6 | YouTube、Bilibili 與 generic provider 維持網站品質矩陣；Bilibili 彈幕 XML／ASS／MKV 流程保留，未驗證網站不宣稱下載支援。 |
| 5.7 | 長影片／BGM 使用章節、描述時間碼或有界音訊斷點產生草案；必須先試聽切點、手動調整並確認命名才加入佇列。 |
| 5.8 | 搜尋控制改為兩列響應式配置，保留深色介面、背景圖片、低閒置刷新、縮圖取消與鍵盤操作。 |
| 5.9 | 新增只讀正式版候選評估器；檢查品質、升級／回退、P0／P1、Ed25519 與 Authenticode，只提示、不簽署、不包裝、不上傳 Stable。 |

上述更新會隨開發版 6.0 完整附件一併交付。
