# MediaManager 3.1～4.0 更新計畫（提案）

此文件是 3.0 完成後的可修改提案，不代表功能已開始製作。排序以相依性、資料
安全與維護成本為準。

| 版本 | 主題 | 建議範圍 | 完成門檻 |
| --- | --- | --- | --- |
| 3.1 | UI 效能與一致性 | **已完成：**下載、轉換與轉錄工作表跳過未變更重繪並自適應刷新 | 對應回歸通過並封裝於 `Version/3.1` |
| 3.2 | 下載可靠性 | **已完成：**輸出路徑及 256 MiB 磁碟保留空間預檢，沿用原子佇列與完成摘要 | 對應回歸通過並封裝於 `Version/3.2` |
| 3.3 | YouTube 音樂工作流 | **已完成：**搜尋、歷史、相似、替代與試聽流程整合；新增本機偏好建議 | 不自動播放或背景搜尋；封裝於 `Version/3.3` |
| 3.4 | 長音訊智慧分割 | **已完成：**既有章節／靜音候選增加短片段、低信心、間隙與名稱檢查 | 永遠先預覽；來源不修改；封裝於 `Version/3.4` |
| 3.5 | 多站 MOD 品質 | **已完成：**Bilibili 彈幕能力與六個通用網站家族的離線品質稽核 | 政策界線必須明列；封裝於 `Version/3.5` |
| 3.6 | 媒體庫體驗 | 大型媒體庫索引優化、封面管理、重複項比較、播放清單改善 | 10 萬筆索引有基準；檔案變更皆可預覽或回復 |
| 3.7 | 轉換與字幕 | 轉換預設管理、硬體能力偵測、字幕／彈幕預覽、失敗回退說明 | 不覆蓋原檔；硬體路徑失敗能安全回 CPU |
| 3.8 | MOD 生態 v2 | SDK 測試沙盒、契約相容報告、離線簽署目錄、可選無廣告 catalog | 第三方 MOD 可完整建立、驗證、更新、回復與移除 |
| 3.9 | 4.0 凍結 | 資料遷移、效能、無障礙、SBOM、依賴授權及發行候選稽核 | 無重大安全、資料遺失、移轉或封裝問題 |
| 4.0 | 長期支援基準 | 穩定契約 v2、文件化相容週期、正式簽署核心與可回復升級 | 正式雙重簽署；3.x 升級／回復與可攜模式全部通過 |

不建議放入 4.0 前主線：雲端帳號強制綁定、廣告、遙測、內建瀏覽器繞過限制、
未授權動畫瘋擷取、Meta 私人內容擷取，以及同時加入大量未驗證網站。
