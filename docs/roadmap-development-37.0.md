# Development 37.0 版本計畫

> 重新設計狀態（2026-07-18）：`SUPERSEDED`。本文件以下內容只保留歷史草案；37.0 的唯一執行定義與優先序改由 [34.0～40.0 主計畫](roadmap-development-34.0-40.0.md) 管理。

歷史草案主題（`DO NOT EXECUTE`）：37.0 原聚焦「下載任務可觀測性、完成通知與可恢復輸出」；內容未依新優先序重寫，不得作為目前規格。

## P0：任務安全

- **任務狀態機**：等待、處理中、暫停、取消、完成、失敗與阻擋狀態只能按合法轉移更新。
- **原子輸出**：暫存檔、部分輸出、取消清理與既有檔案保護維持可恢復且不覆寫。
- **重啟策略**：乾淨啟動不得自動恢復未授權工作；恢復前顯示工作與輸出檢查結果。
- **驗證證據**：Ruff、回歸測試、網站／MOD／版本／文字污染稽核保持可重現；UserData 不進版本雜湊。

## P1：工作區體驗

- YouTube／Music、Bilibili、動畫瘋、MEGA 與 Facebook 的佇列顯示統一進度、速度、剩餘時間與錯誤摘要。
- 完成、失敗、取消與阻擋通知可關閉、可追蹤且不重複彈出；試聽／預覽停止後釋放資源。
- 批量下載支援全選、取消選取、清除已完成與保留失敗項，並顯示實際選取數。
- 依賴缺失、外部 403／Cloudflare 與安全阻擋提供可操作但不繞過限制的說明。

## P2：條件式候選

- 下載工作報告、重試摘要與診斷匯出維持使用者觸發、欄位脫敏與大小上限。
- 新網站子 MOD、Gopeed、P2P／Torrent 與通用解析器維持候選，不進入核心。
- 參考軟體的並行與排程能力只記錄差異，不自動提高本機資源上限。

## 每輪純規劃驗收

1. 唯讀掃描工作樹、文件、參考清單、版本與 MOD 狀態。
2. 建立或補充下一個未完成版本計畫；若沒有新差異，記錄原因與下一輪候選。
3. 只記錄唯讀稽核結果，不修改程式碼、使用者資料或版本產物。
4. 外部網站限制、正式簽章缺失與歷史 UserData 項目維持為阻擋項。

37.0 不建立 Stable 或 Testing 產物；發布需另行取得使用者明確決定。
