# Development 36.0 版本計畫

> 重新設計狀態（2026-07-18）：`SUPERSEDED`。本文件以下內容只保留歷史草案；36.0 的唯一執行定義與優先序改由 [34.0～40.0 主計畫](roadmap-development-34.0-40.0.md) 管理。

歷史草案主題（`DO NOT EXECUTE`）：36.0 原聚焦「可測試的搜尋歷史、失效項目替代與診斷匯出」；內容未依新優先序重寫，不得作為目前規格。

## P0：資料與安全界線

- **歷史資料上限**：搜尋與下載歷史採固定筆數、欄位與檔案大小上限，原子寫入並拒絕不安全路徑。
- **失效項目替代**：替代搜尋只能由使用者觸發，顯示候選來源與相似度，不自動替換原工作或下載。
- **診斷匯出**：匯出前清除 Cookie、權杖、私人路徑與媒體內容，只保留版本、錯誤類別與可重現步驟。
- **驗證證據**：維持 Ruff、回歸測試、網站／MOD／版本／文字污染稽核；UserData 不納入版本雜湊。

## P1：搜尋與 UI

- YouTube／Music、Bilibili 與動畫瘋歷史顯示網域、標題、縮圖、時間與狀態，避免跨 MOD 混淆。
- 搜尋結果提供「重新搜尋」「替代候選」「移除紀錄」等明確操作與取消狀態。
- MEGA 與社群官方頁只記錄網址識別及說明，不保存需要登入的內容或秘密資料。
- 四語言介面對歷史、替代搜尋與診斷狀態使用核心提供的翻譯鍵。

## P2：條件式候選

- 低資源相似搜尋、語言偏好與音樂類別統計維持離線聚合，需使用者啟用且可清除。
- 新網站、Gopeed、P2P／Torrent 與通用解析器維持候選，不進入核心。
- 參考軟體的雲端同步與帳號功能不納入本機優先版本。

## 每輪純規劃驗收

1. 唯讀掃描工作樹、文件、參考清單、版本與 MOD 狀態。
2. 建立或補充下一個未完成版本計畫；若沒有新差異，記錄原因與下一輪候選。
3. 只記錄唯讀稽核結果，不修改程式碼、使用者資料或版本產物。
4. 外部網站限制、正式簽章缺失與歷史 UserData 項目維持為阻擋項。

36.0 不建立 Stable 或 Testing 產物；發布需另行取得使用者明確決定。
