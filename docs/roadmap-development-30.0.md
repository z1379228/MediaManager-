# Development 30.0 版本計畫

狀態：規劃中。29.0 已完成社群官方子網域路由補強；30.0 以「可驗證的網域覆蓋、穩定下載佇列與可回溯診斷」為主，不把未簽署或需繞過網站限制的解析器加入核心。

## P0：發布前必須完成

- **網域路由矩陣**：YouTube／Music、Bilibili、動畫瘋、Facebook、MEGA、Instagram、Threads、X/Twitter 的官方主機與錯誤／仿冒 URL 回歸測試保持完整。
- **下載佇列穩定性**：取消、重試、重啟後不自動恢復未授權工作；暫存檔與原子輸出在 Windows 短暫檔案鎖下可安全重試。
- **MOD 父子狀態**：父 MOD 關閉時子 MOD 不可啟用；不可用、製作中、停用與已啟用狀態在四語言介面一致。
- **版本證據**：Ruff、完整測試、網站／MOD 稽核、版本雜湊與 copied-folder smoke 必須可重現。

## P1：主要功能優化

- YouTube／Music 與 Bilibili 工作區縮圖、預覽、批量佇列狀態的增量更新與錯誤分類。
- 動畫瘋官方目錄、集數、離線索引與使用者自行匯入影片／字幕的完整驗證流程。
- MEGA 檔案／資料夾／壓縮檔／文件類型分流與缺少 MEGAcmd 的明確診斷。
- Instagram／Threads／X 官方頁面與資料匯出入口的子 MOD 狀態同步；不宣稱可繞過登入或存取限制。

## P2：條件式候選

- 只有在取得官方公開、非 DRM、非登入的媒體端點後，才評估動畫瘋直接離線下載。
- Gopeed、P2P／Torrent 與新的通用解析器維持獨立安全邊界，不進入核心。
- 低資源搜尋歷史、失效影片替代搜尋與可匯出的診斷報告。

## 每輪循環驗收

1. 先更新本文件或對應 release 區間的進度紀錄。
2. 只實作一項可離線驗證的低風險變更。
3. 執行相關測試、Ruff、網站品質、MOD 群組與版本稽核。
4. 測試失敗時記錄原因並修正；外部網站限制則保留為阻擋項。

30.0 不建立 Stable 版本；Testing 產物需在 P0 完成並通過 copied-folder smoke 後，另行建立測試版候選。
