# MediaManager 3.6–4.0 更新摘要

## 3.6 搜尋 MOD 合約

- 新增 `SearchCapabilityV2`，網站、內容類別、分頁、預覽能力由 MOD 明確宣告。
- 搜尋核心不再假設所有 MOD 都具有相同功能。

## 3.7 聯合搜尋

- 可同時路由最多 16 個搜尋 MOD、回傳最多 50 筆結果。
- 依影片識別碼或移除追蹤參數後的網址去重。
- 單一 MOD 錯誤只列入失敗資訊，不中斷其他 MOD 搜尋。

## 3.8 下載能力協商

- 新增 `DownloadCapabilityV2`，格式、字幕、彈幕、播放清單、分段與續傳能力均可查詢。
- 工作送出前採 fail-closed 驗證，避免把不支援的選項交給外部 MOD。

## 3.9 可靠度封版

- 新增可重試錯誤的 2、8、30 秒退避決策，上限預設三次。
- 永久失效、登入限制與內容移除不自動重試。
- 內建 YouTube、通用 yt-dlp、Bilibili MOD 已提供能力宣告。

## 4.0 穩定合約

- 搜尋與下載 MOD 介面升級為 v2，既有單一 YouTube 搜尋 API 仍保持相容。
- 預設介面仍只顯示必要功能；外部網站由 MOD 啟用狀態決定是否參與。
- 搜尋筆數、MOD 數量、批次大小與游標長度均有資源上限。

安全邊界不變：不繞過 DRM、付費、登入、地區限制或網站廣告機制。
