# MediaManager 3.6–4.0 技術基礎摘要

> 狀態校正：這些版本建立了 v2 合約與後端元件，不等於所有元件已接入 UI、
> 下載佇列或第三方 MOD manifest。實際 4.0 邊界以 `release-4.0.md` 為準。

## 3.6 搜尋 MOD 合約

- 新增 `SearchCapabilityV2` 與有界欄位驗證。
- 搜尋核心具備能力協商資料結構；目前內建搜尋 provider 的能力仍由核心註冊
  時建立，尚未由 MOD manifest 自行宣告。

## 3.7 聯合搜尋

- 聯合搜尋後端可同時路由最多 16 個搜尋 MOD、回傳最多 50 筆結果。
- 依影片識別碼或移除追蹤參數後的網址去重。
- 單一 MOD 錯誤只列入失敗資訊，不中斷其他 MOD 搜尋。
- 4.0 UI 仍走單一 YouTube 搜尋；來源選擇、健康狀態與跨 MOD 分頁列入 4.1。

## 3.8 下載能力協商

- 新增 `DownloadCapabilityV2`，格式、字幕、彈幕、播放清單、分段、續傳與批次
  上限均可描述。
- 工作送出前已 fail-closed 驗證格式、字幕、彈幕與分段；播放清單、批次上限
  與續傳的完整執行階段 enforcement 列入 4.2。

## 3.9 可靠度封版

- 新增可重試錯誤的 2、8、30 秒退避決策與上限測試。
- 這項決策尚未接入下載佇列；4.0 不宣稱已自動重試。
- 內建 YouTube、通用 yt-dlp、Bilibili MOD 已提供能力宣告。

## 4.0 穩定合約

- 搜尋與下載 v2 合約已建立，既有單一 YouTube 搜尋 API 仍保持相容。
- 預設介面仍只顯示必要功能；跨網站搜尋參與規則尚待 4.1 接入。
- 搜尋筆數、MOD 數量、批次大小與游標長度均有資源上限。

安全邊界不變：不繞過 DRM、付費、登入、地區限制或網站廣告機制。
