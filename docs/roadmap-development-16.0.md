# MediaManager 開發版 16.0 計畫

狀態：16.0 功能與 Development 成品驗證已完成。15.1～15.9 作為整合節點記錄，
沒有建立獨立附件；16.0 已建立新的 Development 成品並真正讀取封裝內 provider。
本文件不授權建立 Testing 或 Stable，實際完成內容記錄於
[`release-15.0-16.0.md`](release-15.0-16.0.md)，且不得覆寫既有版本資料夾。

## 大版本主題

16.0 只處理三個相互關聯的主題：**網域專屬搜尋、父子 MOD 適配、下載工作流可靠性**。
核心不新增網站解析器；每個網域仍由父 MOD 擁有工作區，搜尋、批量、預覽、彈幕或
離線索引由該父 MOD 下的子 MOD 個別啟用。

所有網站流程共用七個可驗收階段：網址辨識、內容發現、資訊分析、有限預覽、加入
佇列、取消／重試、完成驗證。網站不具備某階段時必須顯示明確原因，不借用 YouTube
控制，也不回退到跨網域 generic provider 假裝成功。

## 動畫瘋參考模板

FileCentipede 與 iMyFone TopClipper 只作為互動流程參考，不成為相依套件，也不代表
MediaManager 支援其宣稱的全部網站或能力。

| 參考 | 可採用模式 | 動畫瘋套用方式 | 明確不採用 |
| --- | --- | --- | --- |
| FileCentipede | 任務詳細資料、佇列、暫停／續傳、動態狀態與完成統計 | 官方作品／單集辨識後，顯示目前階段、失敗原因、本機索引工作與使用者匯入媒體的可取消進度 | 加密串流擷取、Cookie／header 接管、常駐瀏覽器監控、廣告或存取限制規避 |
| iMyFone TopClipper | 貼入網址後才亮起下一步、解析完成後選擇格式／位置、下載清單與完成後預覽 | 動畫瘋只採「貼官方網址 → 辨識作品／單集 → 顯示允許動作 → 本機索引／媒體預覽」的分步 UI | 私人／會員內容下載、去浮水印、地區限制規避、動畫瘋串流解析或自動跳廣告 |

動畫瘋在 16.0 仍只讀取公開目錄、作品、封面與集數資料；遇到 Cloudflare、登入、
地區、廣告或播放器限制時必須停止自動解析並顯示官方頁備援。使用者自行取得的本機
媒體可匯入離線索引，但不得把索引描述成已下載的動畫影片。

## 下載類型參考模板

iDownerGo 加入一般下載工作區的流程參考，不綁定動畫瘋，也不成為執行相依套件。

| 參考 | 可採用模式 | MediaManager 對應 | 明確不採用 |
| --- | --- | --- | --- |
| iDownerGo | 單一網址、多網址、播放清單、格式／畫質／位置、下載中／已完成、全部暫停與續傳 | 網域辨識後顯示正確工作區；播放清單先展開且預設不勾選；使用者確認格式、輸出與項目後才原子加入佇列 | 分析後自動開跑、Cookie 私人影片、內建瀏覽器接管、代理帳密、受保護串流或跨站萬用解析承諾 |

這個模板不新增「通用網站」主介面。單一網址、多網址與播放清單只是輸入類型，仍由
網址所屬父 MOD 擁有解析與下載能力；不同網域不共享網站專屬按鈕。

## 本機廣告段落剪除

16.0 候選 `media-ad-trim` 是 `media-convert` 下預設關閉的本機子 MOD，只處理使用者
明確選取且有權處理的本機媒體：

1. 使用者以時間軸新增一個或多個「要移除的片段」，每段有開始、結束與可選標籤。
2. 介面提供切點前後的有限預覽；未完成預覽與確認時不能開始輸出。
3. 輸出使用新的檔名與 `.part` 暫存，不覆寫原檔；取消、空間不足或 FFmpeg 失敗時
   清除未完成檔。
4. 第一階段為了準確處理任意切點，明確使用 H.264/AAC 重新編碼；介面顯示輸出策略，
   不把近似關鍵影格的串流複製誤稱為精確剪除。
5. 第一階段只支援手動片段；靜音、黑畫面或章節只能作為候選標記，不能自動刪除。

這個 MOD 不接收網站網址、不分析線上播放器、不移除網站廣告、不跳過動畫瘋倒數，
也不規避 DRM、登入、付費、地區或其他存取控制。MediaManager 主程式本身仍維持
零廣告。

## 優先級與刪減

1. P0：15.1 修正版、動畫瘋集數 UI、YouTube、Bilibili 與下載取消／重啟政策。
2. P1：社群網址、MEGA、搜尋結果契約與第三方網站 MOD fixture。
3. P2：瀏覽器擴充、剪貼簿監控、排程、P2P、遠端控制及新網站；不進入 16.0。

16.0 不新增第五種語言、不加入廣告、遙測、雲端帳號、強制登入、任意腳本、常駐
背景推薦或大型快取。未經正式簽署身分與使用者明確確認，不建立 Stable。

## 驗證與版本規則

- 每個修正版建立新的 `Version/Development/X.Y`，不得替換既有資料夾。
- 次版本只寫入 `release-15.0-16.0.md` 的對應分節；16.0 才預設建立完整附件。
- 每個網站區塊至少包含路由、provider、工作區、父子 MOD、四語言及失敗案例測試。
- Self Check 維持零網路、零 provider、零暫存；手動網路 smoke 使用獨立 schema 2 JSON。
- 即將公開的 EXE、wheel、`release-info.json` 與 `SHA256SUMS.txt` 上傳前後都核對雜湊。
