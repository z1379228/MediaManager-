# Development 42.0 版本計畫

> 收斂狀態（2026-07-18）：`SUPERSEDED BY FIX-34-02`。本文件以下內容為歷史規劃；唯一 canonical 計畫為 `roadmap-development-34.0-40.0.md`。

歷史狀態（封存時）：規劃中。42.0 作為本輪 12 個開發大版本計畫的收束版，聚焦「計畫總結、阻擋項收斂與下一階段決策」，不自動建立 Stable 或 Testing 產物。

## P0：總結與決策門檻

- **計畫追蹤**：彙整 31.0～42.0 各版 P0/P1/P2 完成、未完成、重複與阻擋項，保留來源與驗收證據。
- **安全阻擋**：正式簽章缺失、外部網站 403／Cloudflare／登入／DRM 限制與歷史 UserData 稽核問題維持明確阻擋，不降低條件。
- **發布決策**：只有使用者明確同意且 P0、Ruff、完整測試、網站／MOD／版本／文字污染與 copied-folder smoke 全部通過，才可建立測試版候選。
- **資料保留**：版本附件與 SHA-256 不刪除；UserData、Cookie、權杖與使用者下載資料不納入計畫或雜湊。

## P1：下一階段候選

- 依網域整理 YouTube／Music、Bilibili、動畫瘋、MEGA、Facebook、Instagram、Threads、X/Twitter 的實際可用功能與缺口。
- 依 MOD 父子狀態、語言契約、依賴快照、搜尋歷史、下載佇列與 UI 可存取性建立優先順序。
- 將參考軟體差異轉為可測試的小項目，維持核心精簡與外部 MOD 安全邊界。
- 產生下一輪版本計畫前，先等待使用者確認是否進入實作／打包階段。

## P2：條件式候選

- 新網站子 MOD、Gopeed、P2P／Torrent、通用解析器與雲端同步只保留規格與安全審查項。
- 自動更新、遠端稽核與發布服務不在本輪排程中執行。

## 每輪純規劃驗收

1. 唯讀掃描工作樹、文件、參考清單、版本與 MOD 狀態。
2. 完成 31.0～42.0 計畫彙整；不重複建立同一版本文件。
3. 只記錄唯讀稽核結果，不修改程式碼、使用者資料或版本產物。
4. 本輪完成後停止 12 版規劃排程，等待使用者決定下一階段。

42.0 不建立 Stable 或 Testing 產物。
