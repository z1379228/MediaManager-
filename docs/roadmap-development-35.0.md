# Development 35.0 版本計畫

> 重新設計狀態（2026-07-18）：`SUPERSEDED`。本文件以下內容只保留歷史草案；35.0 的唯一執行定義與優先序改由 [34.0～40.0 主計畫](roadmap-development-34.0-40.0.md) 管理。

歷史草案主題（`DO NOT EXECUTE`）：35.0 原聚焦「跨 MOD 契約、父子生命週期與可回溯資料格式」；內容未依新優先序重寫，不得作為目前規格。

## P0：契約與生命週期

- **契約版本**：搜尋結果、歷史、下載任務、縮圖與錯誤分類使用明確 schema 版本，未知欄位採安全忽略。
- **父子 MOD 生命週期**：安裝、停用、更新、回滾與移除維持父子順序；父 MOD 關閉時子 MOD 不得啟動。
- **資料遷移**：設定、歷史與佇列採原子寫入，可回復且不覆寫使用者資料；遷移失敗保留明確原因。
- **證據完整性**：Ruff、回歸測試、網站／MOD／版本／污染稽核維持可重現，UserData 不進版本雜湊。

## P1：功能協調

- YouTube／Music、Bilibili、動畫瘋、MEGA 與社群官方頁的父／子 MOD 契約文件化並補足錯誤狀態。
- 下載佇列與搜尋歷史共用工作識別、取消／重試與匯出欄位，避免跨區域 UI 不一致。
- 依賴快照與可選工具狀態提供版本、路徑、缺少原因與安全手動安裝說明。
- UI 語言由核心綁定四種語言，MOD 只讀取核心語言，不允許單一 MOD 破壞整體介面。

## P2：條件式候選

- 新網站子 MOD、Gopeed、P2P／Torrent 與通用解析器先驗證 manifest、權限與簽章，再考慮候選整合。
- 歷史失效項目替代搜尋與診斷匯出維持使用者觸發、數量上限與可取消。
- 參考軟體的資料庫同步與雲端功能不納入本機核心，除非另有明確授權。

## 每輪純規劃驗收

1. 唯讀掃描工作樹、文件、參考清單、版本與 MOD 狀態。
2. 建立或補充下一個未完成版本計畫；若沒有新差異，記錄原因與下一輪候選。
3. 只記錄唯讀稽核結果，不修改程式碼、使用者資料或版本產物。
4. 外部網站限制、正式簽章缺失與歷史 UserData 項目維持為阻擋項。

35.0 不建立 Stable 或 Testing 產物；發布需另行取得使用者明確決定。
