# 開發版 27.0～28.0 計畫

本文件集中記錄 27.0 到 28.0 的開發內容。28.0 仍是 Development 軌，
不代表正式版，也不建立 Stable 附件。開發版不因功能數量而降低 SAFE_MODE、
MOD 完整性驗證或官方網域限制。

## 27.0 基線

- 站點 MOD 已分開 YouTube、Bilibili、動畫瘋、MEGA、Facebook 與社群官方頁面。
- 動畫瘋已有官方目錄、分類／關鍵字搜尋、封面、集數導覽、Cloudflare 手動驗證、
  軟體內官方頁、觀看紀錄、離線索引、使用者控制的下一集與 SHA-256 驗證。
- 動畫瘋離線索引支援安全名稱前綴／後綴、使用者本機媒體匯入與取消清理。
- MOD 四語系、站點矩陣、內建雜湊、Ruff、Pytest 與版本稽核均已納入驗證。

## 28.0 目標

28.0 的重點是「可用性封裝與網域分流」，不是把受限制網站的串流擷取硬塞進核心。
每個網域維持父 MOD／子 MOD 結構，未啟用的子 MOD 不建立背景程序，也不在乾淨啟動
時自動開始下載。

## 28.0 開發起始

- 核心版本切換為 `28.0.0`，Development 世代切換為 `28.0`；27.0 產物不覆寫。
- 動畫瘋工作區新增上一次搜尋／集數讀取的明確重試按鈕，依主 MOD、子 MOD、忙碌
  狀態與上一個操作路徑控制可用性。
- 28.0 尚未建立 Testing／Stable 產物，待 P0 完成並通過 copied-folder、依賴與
  發行前驗證後再評估。

## P0：必須完成

### 動畫瘋 MOD

- 完成官方目錄 → 作品 → 集數 → 官方頁／本機離線索引的單一路徑 UI。
- 維持官方 `ani.gamer.com.tw` 白名單與 Cloudflare 手動驗證提示。
- 觀看紀錄支援清除／匯出，清除紀錄不得刪除離線媒體。
- 只有在確認官方提供「非登入、非 Cookie、非 DRM、非付費限制」的下載來源後，
  才評估接入可取消下載；未確認前維持「匯入使用者已取得的本機媒體」流程。
- 補齊主 MOD 關閉時的子 MOD 狀態、按鈕可用性與錯誤提示測試。

### 核心與 MOD 安全

- 所有新增 JSON、manifest、locale、site-matrix 變更同步 SHA-256。
- 外部 URL、檔案、封面、歷史紀錄維持大小上限、官方網域驗證與原子寫入。
- 啟動時不得自動恢復未完成下載；恢復操作必須由使用者明確觸發。
- 版本建立前執行 Ruff、完整 Pytest、MOD 群組稽核、站點矩陣稽核、版本雜湊稽核。

## P1：主要功能優化

### YouTube／Music YouTube

- 單一影片與播放清單明確分流，單一影片隱藏不可用的展開按鈕。
- 搜尋／播放清單結果補齊縮圖、音訊試聽、影片預覽、停止按鈕與錯誤狀態。
- 批量下載提供全選、取消選取、取消任務、清除已完成項目與逐項重試。
- `music.youtube.com`、`youtube.com`、`youtu.be` 與官方子網域維持同一父 MOD，
  搜尋、批量下載、試聽／試看以子 MOD 分開。

### Bilibili

- 搜尋結果補齊縮圖、UP 主資訊與分集名稱。
- 影片與番劇分開顯示，彈幕 XML／ASS／可選 MKV 封裝維持獨立子 MOD。
- 多段影片下載前顯示段落與合併狀態，不把未命名分段直接送入下載佇列。

### MEGA 與社群頁面

- MEGA 先判定影片／壓縮檔／文件，再顯示對應下載流程；不與 YouTube 基礎 UI 混用。
- Facebook 只處理使用者提供的官方媒體網址；Instagram、Threads、X 維持官方頁面／
  資料匯出入口，不製作登入或限制繞過。

### UI 與語言

- 主 MOD 開啟後才顯示子 MOD，子 MOD 關閉時按鈕呈現明確不可用原因。
- 四語系鍵值必須完整一致；新增 UI 不得讓 940×620 最小視窗出現水平破圖。
- 所有試聽／預覽必須有停止或取消操作，不能留下背景播放工作。

## P2：條件式候選

- 若官方提供公開匯出格式，才評估動畫瘋留言／彈幕匯出子 MOD。
- 若取得合法且不受 DRM／登入限制的來源，才評估動畫瘋分段、重試與下載佇列。
- 評估 Gopeed、P2P、雲端與遠端通知時，先做權限、憑證與資料外傳審查。
- 不把 Cookie 自動更新、代理／鏈式代理、FTP Dashboard、廣告跳過或年齡／登入
  自動化列為功能目標。

2. Ruff、完整 Pytest、MOD／站點／版本稽核全部通過。

## 28.0.x 搜尋歷史與失效影片替代

- YouTube History 選單新增已選取影片清單；每筆歷史影片可開啟原始網址，或啟動受控的替代搜尋。
- 替代搜尋沿用 YouTube Recovery 與 YouTube Search MOD，只依標題、作者、語言及分類產生候選，不會自動替換網址或自動下載。
- 歷史選取項目依 `video_id` 去重並受數量上限限制；不保存 Cookie、登入狀態、串流網址或頁面內容。
- 無歷史搜尋字串時仍可顯示歷史影片；候選結果需由使用者選取後，才可帶入下載工作區。

## 28.0.x P0／P1／P2 開發排序

本節是目前工作佇列，不是可啟用 MOD 清單。P0 先處理會阻擋日常使用或正式候選
驗收的回歸；P1 收斂網站邊界與可選依賴；P2 維持候選，不因排程而降低 SAFE_MODE、
簽章、權限或網站存取限制。實際執行順序固定為 P0 → P1 → P2，同一層再依工作項
ID 排序，以便 CI、UI 自檢與文件產生穩定結果。

| 優先級 | 工作項 | 範圍 | 狀態 |
| --- | --- | --- | --- |
| P0 | YouTube／Music 回歸 | 搜尋、清單、縮圖、試聽／停止、批量、暫停／取消／重啟 | 已完成離線驗證 |
| P0 | Bilibili 回歸 | 搜尋、UP 主／分 P、縮圖、批量、彈幕子 MOD | 已完成離線驗證 |
| P0 | 動畫瘋流程 | 近期熱播、新上架、分類、作品與集數、403／Cloudflare 備援 | 已完成離線驗證 |
| P0 | 語言與父子 MOD 契約 | 四語言、父關閉時隱藏／停用子 MOD | 已完成離線驗證 |
| P1 | MEGA 執行邊界 | 檔案／壓縮檔／文件／資料夾與 MEGAcmd 診斷 | 已完成離線驗證 |
| P1 | Facebook 公開網址 | 只貼網址、失敗分類與安全邊界 | 已完成離線驗證 |
| P1 | Generic yt-dlp 矩陣 | 只宣稱有 fixture／smoke 證據的網站 | 已完成離線驗證 |
| P1 | Speech-to-Text 依賴 | whisper-cli／模型匯入、缺失診斷與不自動下載 | 已完成離線驗證 |
| P2 | Gopeed Bridge | 本機 API 授權與 token 生命週期 | 安全基線完成／未啟用 |

上述排序已同步至 `trusted_ui.planned_mod_catalog` 的唯讀開發佇列；這些工作項不會
自動出現在 runtime provider registry，也不會新增啟用按鈕。每一項完成後才可依
manifest、完整性、契約與回歸測試決定是否轉為現有父／子 MOD 的可用功能。

### P0 完成證據

- YouTube／Music、Bilibili、動畫瘋工作區與四語言／父子 MOD 契約的 P0 測試共 98 項通過。
- 下載佇列的暫停、取消、重啟後保持暫停與使用者明確恢復才執行，沿用既有回歸測試。
- 以上是離線工程驗證；即時網站 provider smoke 仍受上游網路、Cloudflare、服務狀態與
  使用者所在區域影響，不會被標成已通過，也不會以任何方式繞過網站限制。

### P1 完成證據

- MEGA 檔案／資料夾、影片／音訊／圖片／壓縮檔／文件分類、官方 `mega-get` 分流與
  缺少工具時的只讀診斷已通過測試。
- Facebook 僅接受精確的 HTTPS 公開影片網址；縮圖 CDN、格式、失敗訊息與不讀取 Cookie
  邊界已通過測試，登入／權限／地區限制不會被繞過。
- Generic yt-dlp 只保留 Vimeo、Dailymotion、SoundCloud、TikTok、Twitch 的既有矩陣；
  YouTube、Bilibili、Facebook、Instagram、Threads、X 維持各自 MOD 或明確排除。
- Speech-to-Text 只接受使用者匯入且 SHA-256 驗證通過的本機模型；`whisper-cli` 或模型
  缺失時明確顯示不可執行，不自動下載或在乾淨啟動執行。

### P2 安全基線證據

- `core.transfers.policy` 只提供設定驗證，不啟動 Gopeed、P2P、Torrent 或任何背景程序。
- Gopeed 僅接受明確的 localhost endpoint；遠端 endpoint、auto-start、遠端控制與未達
  長度要求的 token 會拒絕，token 不由核心寫入磁碟。
- P2P 必須明確確認合法用途、設定絕對儲存路徑與下載／儲存上限；內建 torrent 搜尋、
  自動開埠與未設定上傳上限的做種一律拒絕。
- P2P 引擎、Gopeed API 呼叫、搜尋、開埠與啟用按鈕仍未註冊為 runtime MOD；正式接入
  前仍需獨立 manifest、簽章、capability 與 provider smoke 證據。
