# MediaManager 開發版 15.0 計畫

狀態：15.0 來源整合、完整測試、Development 成品與 copied-folder 驗證已完成；成品
provider smoke 因固定案例與載入路徑缺陷未放行，後續改由 15.1 修正。本文件不授權
建立 Testing 或 Stable；實際交付證據仍以 `release-14.0-15.0.md` 為準。

## 重新評估結論

14.0 的 825 項測試、完整主視窗、父子 MOD、四語言、版本稽核與 copied-folder
啟動均已通過，因此 15.0 不重寫核心、不新增大量網站，也不再同時大改搜尋、下載、
MOD 安裝與 UI。下一個大版本只處理一個主題：**網站工作流可靠性**。

目標是讓每個網站從「辨識網址／搜尋」到「分析／預覽」再到「排隊／取消／完成」
具有一致、可驗證且不跨網域誤路由的狀態。網站沒有某項能力時，UI 必須隱藏或停用，
不能借用 YouTube 控制，也不能用 generic provider 假裝支援。

## 優化與刪減

- `generic-ytdlp` 維持預設停用 Beta；不因單一 extractor 存在就把網站加入預設 UI。
- `direct-transfer` 與已完成的 `direct-http` 重複，14.1 移除前者；Gopeed 與 P2P 仍為
  P2 候選，不進入 15.0。
- Instagram、Threads 與 X/Twitter 維持官方頁／官方資料匯出，不增加搜尋或自動下載。
- 動畫瘋不製作串流下載、內嵌播放器、跳廣告倒數或限制規避；只保留官方導覽與使用者
  本機媒體封存。
- 語言永久維持 `en`、`ja`、`zh-CN`、`zh-TW`，不增加第五種語言。
- 不加入廣告、遙測、雲端帳號、常駐剪貼簿監控、任意前後處理腳本或背景推薦。

## 版本與發布規則

- 14.1～14.9 只在 `release-14.0-15.0.md` 分節記錄，預設只提供更新簡介。
- 15.0 才建立新的 `Version/Development/15.0` 完整附件；不得替換 14.0。
- Testing 必須使用獨立 1.x 編號並維持 `SAFE_MODE`。Stable 仍受 production
  Ed25519、Authenticode、全綠候選證據及使用者明確確認阻擋。
