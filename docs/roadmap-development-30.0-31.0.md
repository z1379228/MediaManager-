# Development 30.0–31.0 更新計畫

> 收斂狀態（2026-07-18）：`HISTORICAL / SUPERSEDED`。本文件以下「目前執行」等字樣只代表封存當時狀態；唯一 canonical 計畫為 `roadmap-development-34.0-40.0.md`。

本計畫以 Development 30.0 為基線，依 30.1～31.0 逐步強化播放器、動畫瘋 MOD、跨 MOD 契約與發行驗證。所有版本維持 SAFE_MODE，不繞過 DRM、登入、Cookie、Cloudflare、付費或地區限制。

## 30.1：播放器體驗與狀態（目前執行）

### P0

- 統一播放、停止、錯誤、關閉與取消狀態，確保視窗關閉後不殘留背景播放器。
- 播放清單切換時同步標題、目前項目與按鈕啟用狀態。
- 補充本機檔案大小、副檔名與路徑安全檢查。

### P1

- 增加字幕檔選取與匯入提示；字幕仍由使用者明確提供。
- 保存音量與上次播放項目到本機設定。

### P2

- 鍵盤快捷鍵與無障礙名稱整理。

依賴：PySide6 Multimedia、既有 `ani-gamer-player` 契約。

## 30.2：動畫瘋搜尋與集數導覽（目前執行）

### P0

- 搜尋、作品選取、集數載入與縮圖更新採同一 generation token，避免舊結果覆蓋新結果。
- 403／瀏覽器驗證時顯示可理解的官方頁 fallback。

### P1

- 支援搜尋歷史重新載入與失效作品提示。
- 集數列表提供明確載入中、空結果與重試狀態。

### P2

- 優化 940×620 最小視窗與窄螢幕排版。

阻擋：外部官方網站驗證或服務異常時，只能標記阻擋，不降低安全規範。

## 30.3：離線封存與完整性

### P0

- `episode.json`、封面、使用者影片與字幕的原子寫入與 SHA-256 驗證。
- 匯入中斷、取消與部分檔案清理可恢復。

### P1

- 離線資料夾瀏覽、重新驗證與錯誤修復提示。

### P2

- 匯出可讀的封存診斷摘要。

## 30.4：跨 MOD 播放契約

### P0

- 將播放器狀態、錯誤與能力宣告納入核心版本化契約。
- 父 MOD 關閉時自動停用所有子播放器功能。

### P1

- YouTube、Bilibili、AniGamer 使用相同 UI 播放狀態模型，但保留各自網域 MOD。

### P2

- 自檢工具增加播放器能力與依賴快照檢查。

## 30.5：效能與資源管理

### P0

- 限制縮圖、WebEngine 與播放器背景工作數量。
- 關閉工作區時釋放頁面、播放器與網路回呼。

### P1

- 大型搜尋結果採增量更新與可取消載入。

### P2

- 增加 UI 更新耗時與記憶體診斷資訊。

## 30.6：網站 MOD 相容性

### P0

- 檢查 YouTube／Music、Bilibili、AniGamer、Facebook、Instagram、Threads、X、MEGA 父子 MOD 是否可獨立啟用。
- 網域路由與子 MOD capability 不得互相污染。

### P1

- 補足各網站搜尋、純網址下載與預覽能力的 UI 說明。

### P2

- 建立第三方 MOD schema v2 相容性範例。

## 30.7：安全與依賴

### P0

- 依賴鎖定、內建 MOD SHA-256、文字污染與 manifest 稽核納入每次建置。
- 維持外部 MOD 預設停用與必要 capability 最小化。

### P1

- 更新 MOD 開發者指南與錯誤回報格式。

### P2

- 產生可重現的 SBOM 與授權摘要。

## 30.8：UI 整理

### P0

- 修正白屏、破圖、按鈕裁切與深色主題不一致。
- 父 MOD 展開後才顯示子 MOD，且狀態即時同步。

### P1

- 背景圖片、字體大小與密度設定採安全本機設定。

## 30.9：完整回歸與候選評估

- 執行完整 pytest、Ruff、MOD／網站／依賴／版本／文字污染稽核。
- 執行 copied-folder 啟動測試、播放器離線測試與所有父子 MOD 狀態測試。
- 若缺少正式 Ed25519／Authenticode 簽章，僅標記候選，不建立 Stable。

## 31.0：Development 封版

### 31.0 完成狀態（Development）

- 30.1～30.9 的非 UI 功能、回歸測試與稽核已完成；本版建立新的 31.0 開發版資料夾，不覆寫 30.2。
- 31.0 維持 SAFE_MODE；官方頁面若要求瀏覽器驗證，僅提供官方頁面／瀏覽器 fallback，不擷取受限串流。
- 封裝驗收包含 EXE、wheel、release-info.json、SHA256SUMS.txt、copied-folder 啟動與版本稽核。

- 建立全新 `Version/Development/31.0`，不覆寫 30.0。
- 產生 EXE、wheel、SBOM、`release-info.json`、`SHA256SUMS.txt`。
- 驗收條件：所有 P0 通過、完整測試通過、稽核無錯誤、SAFE_MODE 保持啟用。
- 31.0 完成後再評估是否建立 Testing 版本；Stable 仍需使用者明確確認。

## 共通風險與阻擋

- 官方網站驗證、Cloudflare、登入、DRM 或付費限制：只提供官方頁面 fallback。
- Windows 暫存目錄 ACL 或 symlink 能力不足：記錄環境阻擋，不修改安全規範。
- 缺少正式簽章身分：維持 Development／SAFE_MODE，不偽稱正式安全發行版。

## 目前檢查結果（30.1 完成、30.2 待執行）

- 已通過 Ruff、版本稽核（30.0／30.1）、MOD 群組稽核、網站品質稽核、依賴鎖定稽核與文字污染掃描。
- 未開啟 UI 的檢查未發現程式碼錯誤；工作樹仍有未提交的既有修改，未進行 Git 重設或提交。
- Windows 暫存區存在多個 ACL 拒絕存取的舊暫存目錄，屬環境清理阻擋，不判定為程式污染；後續測試使用隔離暫存路徑或提升權限執行。
- 正式 Ed25519／Authenticode 簽章仍缺少，因此所有後續版本維持 Development／SAFE_MODE。
