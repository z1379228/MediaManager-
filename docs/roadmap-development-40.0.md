# Development 40.0 版本計畫

> 重新設計狀態（2026-07-18）：`SUPERSEDED`。本文件以下內容只保留歷史草案；40.0 的唯一執行定義與優先序改由 [34.0～40.0 主計畫](roadmap-development-34.0-40.0.md) 管理。

歷史草案主題（`DO NOT EXECUTE`）：40.0 原聚焦「發布前證據、複製資料夾啟動與回退能力」；內容未依新優先序重寫，不得作為目前規格。

## P0：發布前阻擋項

- **Copied-folder smoke**：測試版候選能在乾淨複製資料夾啟動，UserData 與工具路徑不污染版本目錄。
- **回退驗證**：目前版與上一版可並存、可讀取既有資料並安全回退，不自動刪除使用者資料。
- **簽章狀態**：生產 Ed25519／Authenticode 缺失或 NotSigned 必須標記阻擋，不偽稱正式安全發行版。
- **證據完整**：Ruff、完整測試、網站／MOD／版本／文字污染稽核與 SHA-256 清單可重現；UserData 不進版本雜湊。

## P1：維護與診斷

- 版本資訊、依賴快照、MOD 群組狀態與錯誤日誌在複製資料夾中保持相對路徑與可讀性。
- 下載佇列、歷史資料與設定採版本化遷移，失敗時保留上一份可回復狀態。
- 外部網站 403／Cloudflare、缺少工具與 SAFE_MODE 阻擋在啟動與 MOD 管理頁一致顯示。
- 發布附件只在使用者明確要求且所有 P0 通過後產生；日常排程不打包與不推送。

## P2：條件式候選

- 自動更新提示、差異摘要與回退選擇器先維持規格，不直接下載或執行未知更新程式。
- 新網站、Gopeed、P2P／Torrent 與通用解析器維持候選，不進入核心。
- 雲端發布與簽章服務整合需另行審查憑證、權限與隱私界線。

## 每輪純規劃驗收

1. 唯讀掃描工作樹、文件、參考清單、版本與 MOD 狀態。
2. 建立或補充下一個未完成版本計畫；若沒有新差異，記錄原因與下一輪候選。
3. 只記錄唯讀稽核結果，不修改程式碼、使用者資料或版本產物。
4. 生產簽章缺失、外部網站限制與歷史 UserData 項目維持為阻擋項。

40.0 不建立 Stable 或 Testing 產物；發布需另行取得使用者明確決定。
