4. 只有 P0/P1 驗收完成後才可評估 33.0 Development 打包；不建立 Stable 或 Testing 產物。

## P0：安全與可診斷性

- **依賴狀態一致**：MEGAcmd、yt-dlp EJS、whisper-cli、語音模型與 FFmpeg 的缺少／可用／版本不符狀態在核心與 MOD UI 使用同一份唯讀快照。
- **父子 MOD 權限**：父 MOD 停用時所有子 MOD 隱藏或鎖定；不可用、製作中、停用與啟用狀態不得被按鈕誤判。
- **錯誤可回溯**：Provider、下載佇列、UI 與依賴錯誤保留安全分類、時間與工作識別，不寫入 Cookie、權杖或私人資料。
- **稽核證據**：Ruff、回歸測試、網站／MOD／版本／文字污染掃描結果與阻擋項可重現；UserData 不納入版本雜湊。

## P1：使用流程優化

- 下載工作區統一 YouTube／Music、Bilibili、動畫瘋與 MEGA 的狀態欄位、取消／重試與完成提示。
- 搜尋結果維持縮圖、預覽與網域標籤，避免跨 MOD 誤用或把未支援 URL 當成可下載。
- 依賴提示提供安全的手動安裝說明與檢查結果，不自動執行未知安裝程式。
- 歷史紀錄、失效項目替代搜尋與匯出報告維持使用者觸發、大小上限與原子寫入。

## P2：候選與限制

- 新網站子 MOD、Gopeed、P2P／Torrent 與通用解析器只保留候選規格，需獨立簽章與權限審查。
- 只有官方公開、非登入、非 DRM 端點可驗證時，才考慮新的離線下載功能。
- 參考軟體功能差異僅記錄採用理由，不自動改變安全策略或發布通道。

## 每輪純規劃驗收
