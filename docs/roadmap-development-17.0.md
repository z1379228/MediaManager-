# MediaManager 開發版 17.0 計畫

狀態：規劃中。17.0 延續 16.0 的網域父 MOD／功能子 MOD 架構，但先修復目前來源與
16.0 成品無法一對一重現的發布基線，再逐區塊收斂網站工作流。本文不代表功能已完成，
也不授權建立 Testing 或 Stable；實際完成內容只記錄於
[`release-16.0-17.0.md`](release-16.0-17.0.md)。

## 大版本目標

17.0 聚焦四項可以獨立驗收的成果：

1. **可重現來源與發行**：每個成品必須能對應已提交的 revision、source fingerprint、
   wheel、EXE、版本資訊與 SHA-256，不再由含大量未提交變更的工作區直接發布。
2. **網域工作流完整性**：YouTube／YouTube Music、Bilibili、動畫瘋、Facebook、
   Instagram、Threads、X/Twitter 與 MEGA 維持各自路由、工作區、錯誤與能力邊界。
3. **下載控制可靠性**：使用者確認後才開始；批量選取、暫停、續傳、取消、重試、
   清除、重啟與完成驗證具有一致且可測試的狀態轉換。
4. **可實際開發的第三方 MOD 基線**：schema v2 範本、宣告式 UI、四語言、驗證、
   安裝、停用、更新與除錯流程能由文件和離線 fixture 完整重現。

核心不加入網站專屬解析器。網站父 MOD 未啟用時不顯示其子 MOD；父 MOD 啟用後，
才列出搜尋、批量、預覽、彈幕、官方頁面或資料匯出等適用子 MOD。指定網域失敗時
必須 fail closed，不改查 YouTube 或 generic provider 假裝成功。

## 優先級與刪減

1. **P0**：16.1 可重現基線、CI 覆蓋、YouTube／Bilibili 與下載停止／重啟政策。
2. **P1**：動畫瘋安全導覽、社群／MEGA 邊界、父子 MOD、四語言與第三方範本。
3. **P2**：大型控制器拆分、換行正規化、非阻擋的操作流暢度與診斷改善。

17.0 不加入第五種語言、P2P／Torrent、遠端控制、雲端帳號、強制登入、遙測、廣告、
常駐剪貼簿監控或跨網站自動推薦。`generic-ytdlp` 只保留既有 Beta 相容範圍；沒有獨立
路由、fixture 與 smoke 的新網站不得宣稱支援。

本機 `media-ad-trim` 仍只處理使用者明確選取且有權處理的本機媒體，不接收網站網址、
不跳過動畫瘋倒數、不移除網站廣告，也不規避 DRM、登入、付費、地區或存取控制。

## 驗收矩陣

每個網站區塊至少驗證以下項目：

- 網址分類、父 MOD ownership、子 MOD 顯示與停用後 fail-closed。
- 零筆結果、部分來源失敗、上游暫時錯誤、缺依賴與取消的不同狀態。
- 縮圖來源白名單、回應大小上限、重新導向限制與不可信文字清理。
- 單項與批量輸入、預設選取、工作建立、停止、重試、重啟與完成驗證。
- 四語言資源、940×620 最小尺寸、鍵盤操作與可存取名稱。

共同驗證分成三層：

1. **每次修改**：Ruff、目標 pytest、契約 fixture、Self Check。
2. **次版本候選**：完整 pytest、污染／MOD／網站／依賴稽核及對應 provider smoke。
3. **17.0 封版**：從已提交 revision 建置、copied-folder 啟動、發行 preflight、
   `release-info.json`／`SHA256SUMS.txt` 與上傳前後附件雜湊核對。

## 版本與發布規則

- 每個修正版使用新的 `Version/Development/X.Y`，不得覆寫 16.0 或其他既有資料夾。
- 16.1～17.0 的實際完成內容集中寫入 `release-16.0-17.0.md`，不新增零散 release 文件。
- 開發次版本預設只更新簡介；17.0 才預設建立完整 Development 附件。
- 若次版本需要緊急驗證附件，必須先記錄原因、範圍與雜湊。
- 日常只稽核 current + previous；17.0 大版本封版才執行完整歷史稽核。
- 沒有 production Ed25519／Authenticode、完整驗證及使用者明確決定時，不建立 Stable。
