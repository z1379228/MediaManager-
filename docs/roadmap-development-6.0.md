## 6.0 三項優先目標

1. **搜尋**：來源可選、結果足量、真實分頁、歷史／替代／相似模組分離，且不
   建立常駐推薦或追蹤服務。
2. **MOD 適配**：第三方可用範例、驗證器與遷移報告完成接入；所有外部能力
   預設停用、可取消、有限額並維持 fail-closed。
3. **下載**：實際格式資訊、批次排程、續傳恢復、長影片切割與網站專屬附件
   在同一工作區協作，但網站邏輯仍留在各自 MOD。

## 正式版候選提示條件

同一開發版必須同時符合下列條件，才可提示「可轉正式版」：

- Ruff、完整 Pytest、built-in MOD 完整性、依賴鎖定與 SBOM 稽核全部通過。
- copied-folder GUI、headless、verify、更新與回退測試通過。
- 沒有 P0／P1 已知缺陷，下載工作不會靜默遺失或覆寫使用者檔案。
- 正式 Ed25519 release manifest 與 Windows Authenticode 都有效。
- 功能範圍、隱私、安全限制、網站支援聲明與已知問題均有中文文件。

達標後只回報建議，例如「開發版 5.9 已符合正式版 1.0 候選條件」。只有使用者
明確決定後，才建立 `Version/Stable/1.0`、正式簽署、包裝及上傳。

## 不納入 6.0 核心

- DRM、付費、登入、地區或廣告限制繞過。
- 內建 BitTorrent／Magnet／ed2k、做種或 torrent 搜尋。
- 未經驗證就宣稱 Facebook、Instagram、Threads、MEGA 等網站可下載。
- 雲端帳號同步、遙測、廣告、常駐推薦服務或任意下載後腳本。

這些候選能力若日後評估，仍必須以預設停用的獨立 MOD 處理，不增加乾淨核心
的背景資源占用。
