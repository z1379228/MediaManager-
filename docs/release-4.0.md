# MediaManager 4.0 實際基線

狀態：`4.0.0` 是目前對外提供的未簽署工程基線，但尚未達成正式簽署
preflight。這份文件記錄實際結果，取代
`roadmap-3.1-to-4.0.md` 中 3.6～4.0 的早期提案；各階段細節仍可參考
`release-3.6-to-4.0.md`。

## 已完成

- `SearchCapabilityV2` 與 `DownloadCapabilityV2` 已建立為 4.x 版本化能力合約。
- 聯合搜尋後端可隔離單一 MOD 失敗、限制來源與結果數量，並以穩定識別碼
  去重；目前 UI 仍使用單一 YouTube 搜尋路徑，跨 MOD 介面尚未完成。
- 下載送出前已依 MOD 宣告 fail-closed 驗證格式、字幕、彈幕與分段選項。
  播放清單、`max_batch_size` 與續傳能力仍需補齊執行階段 enforcement。
- 可重試錯誤的有界退避決策與測試已存在，但尚未接入下載佇列自動重試。
- YouTube、通用 yt-dlp 與 Bilibili 下載 MOD 均提供能力宣告；YouTube 搜尋、
  歷史、替代、相似、試聽及長音訊切割仍維持獨立 MOD 邊界。
- 預設 UI 保持精簡；下載 MOD 可個別啟停，但跨網站搜尋啟用狀態尚未完整接入 UI。
- 搜尋結果、游標、批次、佇列、縮圖與外部程序皆有明確資源或逾時上限。

## 4.0 不代表的內容

- 不代表 yt-dlp 列出的每個網站都已完成相容性與政策驗證。
- 不代表 Facebook、Instagram、Threads、動畫瘋或 MEGA 已有自動下載器；目前
  只有經限制的官方頁面入口或候選 MOD 紀錄。
- 不包含內建 BitTorrent、Magnet、ed2k、雲端同步、帳號代管或常駐遠端控制。
- 開發封裝進入 `SAFE_MODE` 是預期行為；正式簽署 manifest、正式 Ed25519
  身分與 Authenticode 仍必須由發行流程另外完成，不能由版本號推定。
- 目前保留的 `Version/4.0` 可通過自身 `SHA256SUMS.txt` inventory 稽核；但用
  已修正三個內建 MOD 預期雜湊的目前 main 回頭執行 `release_preflight`，會因
  版本漂移與缺少正式發布金鑰而阻擋。這不等於舊附件已被竄改，但代表不能把
  舊附件當成目前來源的正式產物；重新封版前必須重建、驗證並重新產生附件。

## 維護規則

- 4.0.x 只接受錯誤、安全、CI、相容性與文件修正，不加入破壞性合約變更。
- 4.x 新功能依 `roadmap-4.0-to-5.0.md` 分段完成；未通過門檻不得標示完成。
- 4.0 後加入的 GitHub CI、自動合併或候選網站目錄，屬於來源庫持續維護，
  除非重新建置並發布，不能倒推為原始 4.0.0 附件已包含。
