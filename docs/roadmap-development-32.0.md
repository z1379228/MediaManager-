# Development 32.0 版本計畫

> 收斂狀態（2026-07-18）：`DONE / SUPERSEDED`。本文件以下內容為歷史規劃；唯一 canonical 計畫為 `roadmap-development-34.0-40.0.md`。

歷史狀態（封存時）：規劃中。32.0 以「跨網域搜尋邊界、錯誤可診斷性與可重現品質證據」為主，延續 SAFE_MODE 與官方公開內容限制。

## P0：邊界與回歸證據

- **跨網域分流**：YouTube／Music、Bilibili、動畫瘋、MEGA、Facebook、Instagram、Threads、X/Twitter 維持獨立父 MOD 與子 MOD 邊界，不將未支援網站誤判為可下載。
- **錯誤分類**：將 URL 不支援、網路暫時失敗、外部 403／Cloudflare、缺少依賴與安全阻擋分開顯示並可追蹤。
- **版本證據**：每次計畫更新保留來源、驗收結果、阻擋項與稽核時間；Ruff、回歸測試、網站／MOD／版本／文字污染稽核維持可重現。
- **使用者資料隔離**：版本雜湊不納入 UserData；完整歷史稽核遇到使用者資料時只報告阻擋，不刪除或放寬安全規則。

## P1：搜尋與工作區優化

- YouTube／Music 與 Bilibili 搜尋結果維持縮圖、預覽、播放清單與批量佇列的單一網域工作區。
- 動畫瘋搜尋、集數索引與離線匯入流程維持官方目錄／使用者自有檔案邊界；403 或 Cloudflare 僅顯示阻擋原因。
- MEGA 檔案、資料夾、壓縮檔與文件類型維持獨立下載分流與依賴診斷。
- 社群官方頁與資料匯出入口只提供網址識別、說明與安全提示，不宣稱繞過登入或限制。

## P2：條件式候選

- 低資源搜尋歷史、失效項目替代搜尋與可匯出診斷報告先維持規劃，需使用者觸發且有明確資料上限。
- Gopeed、P2P／Torrent、通用解析器與新網站子 MOD 維持獨立簽章與權限邊界，不進入核心。
- 參考軟體新增功能只記錄差異與採用理由，不自動改變安全策略或發布通道。

## 每輪純規劃驗收

1. 唯讀掃描工作樹、文件、參考清單、版本與 MOD 狀態。
2. 建立或補充下一個未完成版本計畫；若無新差異，記錄原因與下一輪候選。
3. 可執行唯讀稽核並記錄結果，不修改程式碼、使用者資料或版本產物。
4. 外部網站限制、缺少正式簽章與 UserData 歷史污染維持為阻擋項。

32.0 不建立 Stable 或 Testing 產物；是否發布須另行取得使用者明確決定。
