# Development 29.0–30.0 更新紀錄

## 29.1

- 動畫瘋官方目錄與集數導覽維持官方頁面安全 fallback。
- Development 建置維持 SAFE_MODE，不建立未經確認的 Stable 版本。

## 29.2

- 動畫瘋目錄、集數、封面與離線封存流程補強。
- 保留 `episode.json`、封面、使用者匯入媒體／字幕與 SHA-256 完整性驗證。
- 不擷取串流、不繞過 Cookie、DRM、Cloudflare、登入或付費限制。

## 29.3

- 修正動畫瘋官方嵌入頁面導覽與 Qt WebEngine 版面：主框架只允許官方網域，子資源仍可正常載入。
- 加入官方頁面失敗時的瀏覽器驗證提示與系統瀏覽器 fallback，並改善 700×480 最小嵌入視窗。
- 修正深色 UI 破圖、過濾器按鈕與狀態訊息高度；889 項測試通過。
- 建立 Development 29.3.0 EXE、wheel、`release-info.json` 與 `SHA256SUMS.txt`。

## 29.4

- 新增獨立 `ani-gamer-player` 子 MOD；只有動畫瘋父 MOD 與播放器子 MOD 同時啟用時，才顯示官方嵌入預覽與本機播放控制。
- 參考 PotPlayer／KMPlayer 的基本操作，提供使用者明確選取的本機媒體播放、播放／暫停、停止與音量調整；不擷取動畫瘋串流。
- 官方頁面播放仍受限於官方 WebEngine 頁面與系統瀏覽器 fallback；不繞過登入、Cookie、DRM、Cloudflare 或廣告限制。
- 更新四語系 MOD 群組、父子狀態自檢與完整性雜湊；29.3 保留作為可回退版本。

## 29.5

- 動畫瘋播放器嵌入頁面加入 15 秒載入逾時提示，避免官方驗證或網路異常造成白屏無回應。
- 播放器「停止」只停止本機媒體，不關閉播放器視窗；關閉仍由使用者明確操作。
- 延續官方頁面嵌入、系統瀏覽器 fallback 與本機使用者選取檔案播放；不擷取串流、不繞過 DRM、Cookie、Cloudflare 或登入限制。

## 29.6

- 本機播放器支援一次選取多個使用者媒體檔案，提供上一個、下一個與播放完自動前進。
- 保留影片、音訊與既有字幕匯入流程；播放器不會自行尋找或下載網路媒體。

## 29.7

- 動畫瘋作品與集數搜尋結果依 `video_id` 去重，避免跨頁或重試造成重複列。
- 延續官方網域路由驗證、縮圖載入與瀏覽器驗證 fallback。

## 29.8

- 父 MOD／子 MOD 狀態、播放器按鈕與離線操作維持同一啟用契約。
- 播放器錯誤、停止與關閉流程集中清理，避免背景播放器殘留。

## 29.9

- 完成播放器、動畫瘋搜尋、集數導覽與離線封存回歸檢查。
- 版本／MOD／網站品質／依賴／文字污染稽核列為封版必要條件。

## 30.0

- 建立 Development 30.0 封版，整合 29.5～29.9 修正並保留歷史版本目錄。
- 產生 EXE、wheel、SBOM、`release-info.json` 與 `SHA256SUMS.txt`；未取得正式簽章前維持 SAFE_MODE。

## 30.1

- 本機播放器多檔播放佇列加入檔案存在、一般檔案與大小上限檢查，拒絕不可讀或超過安全上限的檔案。
- 播放器仍只處理使用者明確選取的本機媒體，不解析或下載網站串流。

## 30.2

- 搜尋與集數載入在新結果成功前保留既有有效列表，避免 403 或瀏覽器驗證錯誤造成畫面清空。
- generation token 持續阻擋過期背景回呼覆蓋目前搜尋結果；成功回應才替換列表。
