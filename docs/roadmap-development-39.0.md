# Development 39.0 版本計畫

> 重新設計狀態（2026-07-18）：`SUPERSEDED`。本文件以下內容只保留歷史草案；39.0 的唯一執行定義與優先序改由 [34.0～40.0 主計畫](roadmap-development-34.0-40.0.md) 管理。

歷史草案主題（`DO NOT EXECUTE`）：39.0 原聚焦「可存取性、語言一致性與 MOD 狀態透明度」；內容未依新優先序重寫，不得作為目前規格。

## P0：介面與安全狀態

- **四語言契約**：繁中、簡中、英文、日文共用核心語言鍵；缺少翻譯顯示安全的回退文字，不混入未翻譯亂碼。
- **MOD 狀態透明**：父 MOD、子 MOD、製作中、不可用、停用與已啟用狀態在管理頁與工作區一致。
- **可存取性**：主要按鈕、表格、縮圖、錯誤與阻擋訊息具備可存取名稱、鍵盤順序與明確焦點。
- **驗證證據**：Ruff、回歸測試、網站／MOD／版本／文字污染稽核保持可重現；UserData 不進版本雜湊。

## P1：使用流程

- YouTube／Music、Bilibili、動畫瘋、MEGA 與社群官方頁的父子 MOD 選項使用一致的下拉與勾選模型。
- 搜尋、預覽、試聽、批量下載、取消與錯誤訊息以核心語言顯示，不因網站切換造成 UI 破圖。
- 依賴提示、SAFE_MODE 與外部限制訊息提供四語言的原因與安全下一步。
- 深色背景、可自訂背景圖片與高對比文字維持最小視窗可用，不載入未信任外部樣式。

## P2：條件式候選

- 自訂語言包與 MOD 翻譯擴充先維持規格候選，需核心驗證與完整鍵集合。
- 新網站、Gopeed、P2P／Torrent 與通用解析器維持候選，不進入核心。
- 參考軟體的主題商城、雲端同步與帳號功能不納入本機免費核心。

## 每輪純規劃驗收

1. 唯讀掃描工作樹、文件、參考清單、版本與 MOD 狀態。
2. 建立或補充下一個未完成版本計畫；若沒有新差異，記錄原因與下一輪候選。
3. 只記錄唯讀稽核結果，不修改程式碼、使用者資料或版本產物。
4. 外部網站限制、正式簽章缺失與歷史 UserData 項目維持為阻擋項。

39.0 不建立 Stable 或 Testing 產物；發布需另行取得使用者明確決定。
