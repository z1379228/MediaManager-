# MediaManager 4.1 搜尋清晰度

狀態：來源里程碑完成。此版本依 4.0～5.0 權威路線補齊搜尋接線，不重建或
覆寫 GitHub `v4.0.0` 附件；次版本只提供中文更新說明。

## 已完成

- 搜尋 UI 改用有界聯合搜尋，僅查詢使用者啟用的搜尋 MOD。
- 使用者可選所有已啟用來源或指定來源，並看見正常、錯誤與停用狀態。
- 單一來源失敗會顯示 provider 與原因，不會讓其他來源結果消失。
- 每筆結果保留來源歸屬，顯示縮圖、標題、作者、時長、媒體類型及符合度。
- 搜尋能力由 MOD manifest 的 Search v2 宣告提供；欄位錯誤或 provider ID
  不一致時 fail-closed。
- 常見媒體詞別名與少量明確拼字錯誤在本機有界修正，不建立背景推薦或遙測。
- 結果依標題與作者文字符合度做穩定本機排序，提示中顯示排序理由。
- `generic-ytdlp` 不再誤宣告 Facebook、Instagram 或 Threads 自動下載能力。

## 邊界

- 目前 bundled 搜尋來源仍只有 `youtube-search`；聯合介面是為後續獨立 MOD
  預留，不代表已支援未驗證網站。
- Search v2 的 `pagination=none` 會被忠實呈現，不模擬不存在的下一頁。
- 不執行背景搜尋，不上傳搜尋紀錄，不加入大型圖片快取。
- 4.2 才處理下載前格式、容量、命名與批量預覽；本版本不宣稱已完成下載門檻。

## 驗證

- Ruff 必須通過。
- 完整 Pytest 必須通過；Windows 帳戶無 symlink 權限的兩項測試可保持明確跳過。
- built-in provider manifest 與來源雜湊必須一致。
