# Development 31.0–32.0 更新計畫

> 收斂狀態（2026-07-18）：`HISTORICAL / SUPERSEDED`。本文件以下內容為歷史規劃；目前唯一活動計畫為 Development 34.0。

本區間以 Development 31.0 為基線，規劃 31.1～32.0 的播放器、動畫瘋、網站 MOD 分流、下載恢復與發行驗證。所有版本維持 `SAFE_MODE`；不繞過 DRM、登入、Cookie、Cloudflare、廣告、付費或地區限制，也不擷取受限串流。

## 目前狀態

- 31.0：已完成本機 Development 封裝、copied-folder headless 啟動、完整測試與雜湊稽核。
- 31.1～31.9：已完成播放器邊界、動畫瘋搜尋／封存、網站 MOD 分流、預覽／下載、自檢、UI 與安全稽核；每個次版本均以非 UI 測試驗證。
- 32.0：進行 Development 封裝與最終驗收；建立獨立資料夾，不覆寫既有版本。
- Stable：尚未建立；必須取得使用者明確確認、正式簽章與完整 preflight 才能評估。

## 31.1：播放器 MOD 基礎

### P0

- 建立宣告式播放器 MOD 介面，支援本機影片、字幕與封面，不接受外部 MOD 任意 Qt/HTML/腳本注入。
- 播放來源只允許使用者明確選取的本機檔案；無法播放時顯示可診斷錯誤與官方頁面 fallback。

### P1

- 加入播放佇列、上一個／下一個、停止、鍵盤操作與字幕檔關聯。
- 播放器生命週期與主視窗關閉時確實釋放 WebEngine、Multimedia 與縮圖資源。
- 主視窗建立前顯示一次初始 MOD 選擇，保存後不再重複；子 MOD 依父 MOD 啟用狀態套用。

### P2

- 儲存上次播放位置與音量（僅本機 UserData，提供清除選項）。

## 31.2：動畫瘋官方目錄搜尋

### P0

- 搜尋、近期熱播、新上架、所有動畫、人氣排序與作品選取流程保持在動畫瘋 MOD 內。
- 遇到官方瀏覽器驗證或 403 時保留既有結果，明確標示外部阻擋並提供官方頁面開啟，不繞過驗證。

### P1

- 作品封面、名稱、官方網址、集數列表與歷史紀錄同步；以 `video_id` 去重並避免競態覆蓋。
- 集數只在使用者明確選取後進入離線封存流程。

### P2

- 對公開目錄加入受限大小與短期快取，提供清除快取按鈕。

## 31.3：動畫瘋離線封存

### P0

- 封存 `episode.json`、封面、使用者匯入影片／字幕與 SHA-256；採原子寫入、失敗回復與完整性驗證。
- 不擷取動畫瘋受限串流；若沒有使用者提供的本機媒體，僅保存公開資料與官方連結。

### P1

- 支援批次封存佇列、取消、重試、部分完成狀態與可讀錯誤摘要。
- 匯出／匯入封存索引時驗證路徑、檔案大小與 schema 版本。

### P2

- 提供封存清單篩選、搜尋與過期索引整理，不刪除使用者媒體。

## 31.4：YouTube／Bilibili MOD 分流

### P0

- 父 MOD 只負責網域路由；搜尋、批量下載、試聽／試看、Bilibili 彈幕各由子 MOD 提供，未啟用父 MOD 時不顯示子 MOD。
- YouTube、music.youtube.com、Bilibili 與彈幕 URL 不跨工作區混用；所有不支援網址回報可操作原因。

### P1

- 播放清單預覽縮圖、選取數量、取消／清除列表與工作狀態同步。
- Bilibili 彈幕僅處理使用者可合法取得的公開 XML／ASS 檔案，並提供可選 MKV 封裝。

### P2

- 加入相似搜尋與歷史替代建議，但不自動替換使用者已確認的下載目標。

## 31.5：預覽與媒體轉碼

### P0

- 預覽 MOD 與下載 MOD 分離；預覽失敗不得啟動下載或擷取串流。
- FFmpeg 缺失時顯示依賴提示並維持 SAFE_MODE，允許使用者選擇是否安裝或改用本機播放器。

### P1

- 支援音訊／影片格式、解析度與字幕選項的安全驗證；輸出路徑與檔名限制沿用核心契約。

### P2

- 低資源預覽模式與縮圖降採樣，避免大型清單重繪。

## 31.6：下載恢復與網站類型

### P0

- 下載佇列支援暫停、停止、取消、重啟後不自動續跑；以交易狀態恢復並清理未完成暫存。
- MEGA 依影片／壓縮檔／文件分類，與 YouTube／Bilibili 工作區完全分離。

### P1

- 批次下載輸出命名、磁碟安全保留、重試與完成通知統一；不覆蓋既有檔案。

### P2

- 下載歷史與失效網址提供人工替代搜尋，不自動抓取未授權來源。

## 31.7：依賴與 SAFE_MODE 體驗

### P0

- 啟動自檢顯示 yt-dlp EJS、FFmpeg、Deno、MEGAcmd、whisper-cli 與 speech model 狀態；缺失時只提示，不在背景啟動程序。
- 依賴安裝流程維持來源、雜湊與授權檢查；未驗證套件預設停用。

### P1

- 自檢結果可匯出 JSON；錯誤日誌移除權杖、Cookie、絕對敏感路徑與媒體內容。

### P2

- 提供離線依賴檢查與回復上一版設定。

## 依賴、驗收與阻擋

- 依賴：PySide6、QtWebEngine、FFmpeg（可選）、yt-dlp／EJS（網站 MOD）、Deno（EJS）、MEGAcmd、whisper-cli 與 speech model（各自可選）。
- 每個版本完成 P0 才能進入下一版本；P1／P2 必須有測試或明確標示延期原因。
- 外部網站驗證、Cloudflare、登入、DRM、付費限制或缺少正式簽章時，標記阻擋並維持 SAFE_MODE，不提供繞過方案。
- 任何正式版候選都必須先取得使用者明確確認；本計畫不自動建立 Stable 版本。
