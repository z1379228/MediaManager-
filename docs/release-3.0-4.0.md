# MediaManager 開發版 3.0 至 4.0 更新紀錄

本文件集中記錄 3.x 到 4.0 的實際完成內容。各小版本不再分散成獨立檔案；
歷史驗證數字保留於對應版本段落，未完成的舊提案不視為已交付。

## 3.0

- 下載工作表改為增量刷新，依處理中、閒置及隱藏狀態調整更新頻率。
- 提供 schema v2 MOD 範本、manifest／套件／runtime contract 檢查及中文開發流程。
- 發布依賴與授權清單改為可重現產生；Ruff 與 458 項測試通過。
- 1.0～3.0 共 19 個保留版本完成 wheel、版本號與 SHA-256 稽核。

## 3.1

- 轉換與轉錄表格跳過未變更重繪，降低大型工作清單的 UI 負擔。
- 背景工作依可見性及活動狀態採不同刷新間隔。

## 3.2

- 下載前檢查輸出路徑與 256 MiB 磁碟安全保留。
- 延續原子輸出、取消清理、完成通知與個別工作重試。

## 3.3

- 整合 YouTube 搜尋、歷史、相似、失效替代及可停止試聽。
- 搜尋偏好只在本機保存，不加入遙測或常駐推薦程序。

## 3.4

- 長音訊切割加入過短片段、低信心切點、預設名稱與時間間隙提示。
- 使用者確認切點前不產生最終切割下載。

## 3.5

- 新增離線網站能力稽核，涵蓋六個通用網站家族及 Bilibili 能力。
- Ruff 與 468 項測試通過；1.0～3.5 共 24 個版本完成雜湊稽核。

## 3.6

- 建立 `SearchCapabilityV2` 與有界搜尋欄位驗證。
- 搜尋能力開始由 MOD 宣告，保留舊 YouTube 搜尋相容入口。

## 3.7

- 聯合搜尋最多路由 16 個 MOD、回傳 50 筆結果。
- 依媒體識別碼或移除追蹤參數後的網址去重；單一來源錯誤不影響其他來源。

## 3.8

- 建立 `DownloadCapabilityV2`，宣告格式、字幕、彈幕、播放清單、分段、續傳及批次上限。
- 工作送出前以 fail-closed 方式驗證格式與網站能力。

## 3.9

- 建立 2／8／30 秒有界重試決策，永久錯誤不盲目重試。
- YouTube、generic-ytdlp 與 Bilibili 內建 MOD 加入能力宣告。

## 4.0

- 搜尋與下載 v2 合約成為 4.x 穩定基線，保留舊 API 相容層。
- 聯合搜尋具來源隔離、去重、資源上限與部分失敗資訊。
- 下載送出前驗證格式、字幕、彈幕與分段選項；自動重試尚未接入佇列時不宣稱完成。
- 開發封裝維持 `SAFE_MODE`；不宣稱支援 DRM、付費、登入、地區限制或未驗證網站。

4.0 後的演進接續記錄於 [`release-4.0-5.0.md`](release-4.0-5.0.md)。
