# Development 43.0 版本計畫

狀態：`HISTORICAL / PARTIALLY ABSORBED / REMAINDER CLASSIFIED`。P0 能力契約與部分 P1 離線封存／播放回退
已實作；尚未完成的影片讀取診斷、人工 UI 與候選／外部證據已收斂至唯一活動計畫
`docs/roadmap-development-34.0-40.0.md` 的 `G37-01`／`G38-01`／`G40-01`。尚未接線的
capability-to-button routing 沒有可重現缺口或 failing regression，保留在 deferred backlog，
不宣稱已實作或已轉交上述 Gate。本文件保留歷史
設計背景，不再作為平行執行計畫，也不授權建立 Testing 或 Stable 產物。

## 目標與範圍

- 以父 MOD／子 MOD 能力契約描述各網域的搜尋、預覽、下載、字幕、彈幕與封存能力，避免 UI 顯示與實際能力不一致。
- 播放器 MOD 僅處理使用者明確選取的本機媒體，或嵌入官方公開頁面；官方頁面無法載入時提供可理解的瀏覽器／外部官方頁 fallback。
- 離線封存採可恢復的 manifest、封面／字幕／媒體清單與 SHA-256 驗證，不擷取受限串流。
- 維持 `SAFE_MODE`：不繞過 DRM、登入、Cookie、Cloudflare、廣告、付費或地區限制。

## P0：能力契約與安全邊界

1. **Provider capability schema**：為搜尋、單片、批量、預覽、字幕、彈幕、封存與本機播放定義版本化欄位；未知欄位採拒絕或唯讀降級。
2. **父／子 MOD 一致性**：父 MOD 停用時子 MOD 隱藏或鎖定；啟用、停用與初始化採單一交易，失敗可回復並產生去識別化診斷事件。
3. **輸入與路徑界線**：URL、manifest、字幕、封面與封存路徑設定大小上限、路徑隔離及原子寫入；不把 Cookie、權杖或登入資料寫入日誌與 manifest。
4. **能力矩陣稽核**：補充 MOD／網站稽核，驗證宣告能力與按鈕路由一致，並覆蓋「不可用／製作中／未啟用」狀態。

### P0 驗收

- Ruff、核心／契約／MOD 錯誤路徑測試通過。
- `tools.audit_mod_groups`、`tools.site_quality_audit`、`tools.dependency_lock_audit`、`tools.audit_versions` 與文字污染掃描通過。
- 未簽署 MOD、外部驗證或缺少 FFmpeg／Deno 時維持 SAFE_MODE 並明確標記阻擋。

## P1：播放回退與離線封存

1. **播放器 MOD**：支援本機影片、音訊與字幕選取、播放／暫停／停止、錯誤狀態及最小視窗可用性；不讓外部 MOD 注入任意 Qt／HTML 程式碼。
2. **官方頁面回退**：官方公開頁面可嵌入時顯示載入狀態；若媒體來源不可用，提供瀏覽器驗證或官方頁面開啟按鈕，不嘗試擷取或破解串流。
3. **封存 manifest v1**：保存來源頁、作品／集數識別、檔案角色、語言、時長（若可得）、建立時間與 SHA-256；使用者可重新驗證、匯入或取消，不自動重試受限來源。
4. **下載工作區統一狀態**：沿用現有 site MOD 分流，統一處理中／取消／失敗／重試／完成通知與可存取名稱；跨 MOD 搜尋不得直接共用下載權限。

### P1 驗收

- 本機播放器與 manifest 有離線單元測試、取消／重試／損毀檔案失敗案例。
- 940×620 最小視窗下 UI 不破圖；鍵盤可操作且狀態文字清楚。
- 外部網站需登入、Cloudflare、DRM 或付費內容時，測試只驗證安全錯誤與 fallback，不驗證繞過結果。

## P2：可選擴充與維護性

- 提供第三方 MOD capability 範本、遷移指南與範例測試資料；安裝後預設停用，未簽署套件不得啟動。
- 增加封存報告匯出與歷史失效項目替代搜尋規格，所有搜尋由使用者明確觸發並設大小上限。
- 記錄低成本效能指標（佇列長度、UI 重繪次數、封存驗證耗時），預設不遙測、不上傳。

## 依賴、風險與阻擋

- FFmpeg、Deno、whisper-cli、語音模型與 MEGAcmd 僅作可選依賴；缺少時顯示手動安裝說明，不自動執行未知安裝程式。
- 舊的不可讀暫存目錄仍可能產生 Windows ACL 警告；安全測試入口已隔離 `tmp_path`，
  不需修改既有目錄權限或刪除使用者資料，亦不再視為完整回歸阻擋。
- 未取得生產 Ed25519／Authenticode 簽章前維持 SAFE_MODE，不建立 Stable；未經使用者明確同意不建立 Testing。
- 外部網站內容、驗證流程與版權狀態可能變動；只採官方公開、非登入且可合法驗證的資料。

## 交付順序

1. 完成 P0 契約與稽核，再進行 P1 播放器／封存功能。
2. 以離線測試與 copied-folder headless 啟動驗證每一個可交付節點。
3. 產物、版本號、`release-info.json` 與 `SHA256SUMS.txt` 僅在使用者另行要求時建立；歷史版本不可覆寫。
